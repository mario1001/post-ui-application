This is a simple frontend application for managing posts with an specific API. Using web components (noted as view-* in the project) for exposing the different pages.

For now there is no need for a backend/middleware application (no dealing with complex formats neither data reestructuring). Build with a domain module architecture along with components, services and models inside them. Not using for now the shared module because it is a simple project. There is no environment vars also for the same reason, the API endpoint is served externally without any possible changes.

We are not using Angular material here, not very experienced on it but instead using ng-bootstrap (v5) along its features. The application have its own router along with one specific main module for now.

Another last important note is that the project for now contains css instead of scss/sass tools, maybe it's really a great idea to integrate them instead.
